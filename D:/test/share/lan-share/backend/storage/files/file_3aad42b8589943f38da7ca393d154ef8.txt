# Role
你是一位获得过 Awwwards 奖项的资深前端开发工程师和 UI/UX 设计师，擅长创造极具现代感、视觉冲击力强的网页界面。你非常反感千篇一律的“默认 AI 风格”（如生硬的紫色渐变背景、粗糙的卡片布局）。

# Task
请为我编写一个单页面的“每日天气网站”代码（HTML + CSS + JavaScript）。

# Critical Design Requirements (必须严格遵守)
1. **拒绝平庸设计**：绝对不要使用紫色的默认渐变背景，不要使用标准的 Bootstrap 风格或简陋的卡片。界面必须看起来像是一个专业的商业级产品。
2. **动态实时背景**：
   - 背景必须是全屏的高清真实照片，且根据当前的天气状况动态切换。
   - 逻辑示例：如果是“晴天”，背景展示阳光明媚的风景照；如果是“雨天”，展示雨滴落在窗户或街道的照片；如果是“多云”，展示云层丰富的天空。
   - *技术实现提示*：你可以使用 Unsplash Source API 或类似的免费图库接口，根据天气关键词（如 `sunny`, `rainy`, `cloudy`）动态加载图片 URL。
3. **毛玻璃拟态风格 (Glassmorphism)**：
   - 天气信息卡片必须采用高级的毛玻璃效果。
   - 关键 CSS 属性：`backdrop-filter: blur(15px)`，半透明白色背景 (`rgba(255, 255, 255, 0.15)`)，细腻的白色边框 (`border: 1px solid rgba(255, 255, 255, 0.2)`)，以及柔和的多层阴影以营造悬浮感。
   - 文字颜色需根据背景亮度自动调整（或使用高对比度的白色/深色），确保在复杂背景下依然清晰可读。
4. **排版与布局**：
   - 使用现代无衬线字体（如 Inter, Roboto, 或系统默认美化字体）。
   - 布局居中，留白充足，温度数字要大而醒目，天气图标要精美（可使用 SVG 或高质量图标库）。
   - 添加轻微的入场动画（Fade-in + Slide-up），让页面加载时更有质感。

# Functional Requirements
- 模拟获取当前天气数据（由于没有后端，请在 JS 中模拟一组数据，例如：北京，24°C，晴天，湿度 45%），但代码结构要预留好接入真实 API（如 OpenWeatherMap）的位置。
- 根据模拟的天气状态，自动改变背景图片和界面上的天气描述。



现在，请开始创作这个令人惊艳的天气网站。
